# Project Entry Rules

Before making any change:

1. Read `STATUS.md`.
2. Read every authority file named by `STATUS.md`.
3. Verify the exact repository head.
4. State the current authority, active lane, allowed scope, forbidden changes,
   validation and stop point.
5. Continue only the complete authorised lane.
6. Do not begin a new lane automatically.

Stop only for:

- authority conflict;
- unclear scope;
- unexplained validation failure;
- a forbidden change becoming necessary;
- irreversible action requiring approval;
- genuine human input.
